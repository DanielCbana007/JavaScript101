<h1>¿ Que es JavaScript ?</h1>

## **Introducción al mundo de JavaScript:**

En el ámbito del desarrollo web, JavaScript se posiciona como uno de los lenguajes más recomendados para principiantes, a la vez que se alza como una de las tecnologías más demandadas por la industria TI. Si estás considerando iniciar una carrera en este campo, es probable que te encuentres con esta poderosa herramienta. En ese caso, no es de extrañar que te surjan preguntas como: ¿qué es JavaScript? o ¿para qué sirve?

**JavaScript: El alma de las páginas web dinámicas**

De la mano de herramientas como HTML y CSS, JavaScript desempeña un papel fundamental en el funcionamiento de un sitio web, desde su estructuración hasta su funcionalidad e interacción con el usuario.

**¿Qué es JavaScript?**

JavaScript es un lenguaje de programación ligero, interpretado o compilado justo a tiempo, con funciones de primera clase, utilizado para crear páginas web dinámicas e interactivas. Si bien es más conocido como un lenguaje de scripting para páginas web, también se utiliza en diversos entornos fuera del navegador, como Node.js, Apache CouchDB y Adobe Acrobat.

**Características clave de JavaScript:**

- **Basado en prototipos:** Su estructura se basa en prototipos, lo que facilita la creación y herencia de objetos.
- **Multiparadigma:** Permite emplear diferentes paradigmas de programación, como la orientada a objetos, imperativa y declarativa, ofreciendo flexibilidad a los desarrolladores.
- **De un solo hilo:** Ejecuta las instrucciones de forma secuencial, una a la vez.
- **Dinámico:** No requiere de la declaración previa de tipos de datos, simplificando la escritura del código.
- **Compatible con navegadores modernos:** Funciona a la perfección en los navegadores web actuales, garantizando una amplia cobertura.

**Orígenes de JavaScript: Una breve historia**

Este lenguaje fue creado por Brendan Eich en 1995 con el objetivo de optimizar las páginas web y brindar una experiencia interactiva a los usuarios. Si bien se inspira en otros lenguajes de programación, es importante destacar que, a pesar de la similitud en su nombre, no guarda relación con Java.

**El surgimiento de JavaScript en la era digital:**

JavaScript nació en el apogeo de Internet, cuando las páginas web se limitaban a contenido estático en forma de texto e imágenes. Con la llegada de esta tecnología, se buscaba dinamizar la navegación y facilitar la interacción con el contenido, permitiendo, por ejemplo, completar formularios sin necesidad de recargar la página por completo.

**En resumen, JavaScript se ha convertido en una herramienta indispensable para el desarrollo web moderno, impulsando la creación de páginas web dinámicas, interactivas y atractivas.**

# Interpretación y Compilación en JS

### **Interpretación:**

El proceso de interpretación en JavaScript, en el cual se ven involucrados sus motores (V8 en Chrome, SpiderMonkey en Firefox), históricamente traducía línea por línea el código JavaScript y lo convertía en lenguaje de bajo nivel que la computadora pudiera entender y ejecutar. Sin embargo, los motores modernos utilizan una combinación de interpretación y compilación Just-In-Time (JIT). A diferencia de los lenguajes compilados, donde primero se tiene que compilar el código en un archivo ejecutable, en JavaScript este proceso se da en tiempo de ejecución. Los procesos clave involucrados son:

1. **Parsing (Análisis Sintáctico)**:

    El motor de JavaScript analiza el código JavaScript para entender su sintaxis dividiéndolo en tokens, las unidades básicas del lenguaje, como palabras clave, identificadores, operadores, etc. Una vez separados estos tokens, se utilizan para construir un árbol de sintaxis abstracta (AST, por sus siglas en inglés).

1. **Lexical Analysis (Análisis Léxico)**:

    En este paso, el motor de JavaScript clasifica los tokens según su tipo (por ejemplo, si un token es un identificador o un número).

3. **Creación del Árbol de Sintaxis Abstracta (AST)**:

    Una vez realizados el análisis léxico y sintáctico, se crea un árbol de sintaxis abstracta (AST) que representa la estructura del código JavaScript. El AST es una representación en forma de árbol que organiza las diferentes partes del código, como expresiones, declaraciones y operadores.

4. **Syntax Analysis (Análisis Sintáctico)**:

    El motor de JavaScript utiliza el árbol de sintaxis abstracta (AST) para verificar si el código es sintácticamente correcto. Si hay errores, se lanzan excepciones.

5. **Compilación y Optimización (opcional)**:

    Algunos motores de JavaScript usan métodos de compilación como Just-In-Time (JIT) para optimizar el código antes de ejecutarlo. Durante la compilación, el código JavaScript puede ser traducido a un lenguaje intermedio más eficiente o directamente a lenguaje de máquina para mejorar aún más su rendimiento.

6. **Execution (Ejecución)**:

    Una vez que el código JavaScript ha sido analizado, compilado (si es necesario) y optimizado, el motor de JavaScript ejecuta el código línea por línea. Durante la ejecución, el motor interpreta las instrucciones del código y realiza las operaciones correspondientes, como asignaciones de variables, llamadas a funciones, operaciones aritméticas, etc.

7. **Gestión del Scope y la Memoria**:

    Durante la ejecución, el motor se encarga de gestionar el scope de las variables y la asignación de memoria para los objetos y variables. Esto incluye la creación del scope para las funciones, la resolución de variables y la recolección de basura para liberar la memoria de los objetos que ya no son necesarios.

8. **Garbage Collection (Recolección de Basura)**:

    También realiza una tarea de fondo llamada recolección de basura, que elimina objetos que ya no se utilizan para liberar memoria.

9. **Manejo de Excepciones y Errores**:

    El motor de JavaScript también maneja excepciones y errores que pueden ocurrir durante la ejecución del código. Esto incluye la detección de errores de sintaxis, errores de tiempo de ejecución y excepciones lanzadas por el código.

### **Compilación en JavaScript:**

Históricamente, JavaScript era un lenguaje interpretado, lo que significa que se ejecutaba línea por línea sin una fase de compilación previa. Para mejorar el rendimiento, algunos motores de JavaScript han incorporado técnicas de compilación Just-In-Time (JIT). Este enfoque combina los beneficios de la interpretación y la compilación. Estos son los pasos clave involucrados en la compilación JIT:

1. **Parsing (Análisis Sintáctico):**
    
    Ya tratamos esto anteriormente, pero en resumen, es un proceso donde se realiza un análisis léxico y sintáctico, separando el código en tokens y generando un árbol de sintaxis abstracta (AST).
    
2. **Creación del Árbol de Sintaxis Abstracta (AST):**
    
    El AST es una representación en forma de árbol que organiza las diferentes partes del código (expresiones, declaraciones, operadores, etc.).
    
3. **Bytecode (Código Intermedio):**
    
    En algunos motores, el AST se convierte en bytecode, una forma intermedia de código que es más eficiente y fácil de interpretar que el código fuente original.
    
4. **Compilación Baseline (Inicial):**
    
    En esta etapa, el motor compila el bytecode a código máquina sin muchas optimizaciones antes de ejecutarlo. Esto permite que el código se ejecute rápidamente al inicio, aunque no esté completamente optimizado.
    
5. **Execution (Ejecución):**
    
    El código máquina se ejecuta directamente por la CPU, lo que proporciona un rendimiento más rápido que la interpretación directa del código JavaScript.
    
6. **Profiling (Perfilado):**
    
    Mientras el código se ejecuta, el motor de JavaScript recopila información sobre el comportamiento del código (por ejemplo, qué funciones se llaman con frecuencia, qué tipos de datos se utilizan, etc.).
    
7. **Optimization (Optimización):**
    - Basado en la información recolectada en el proceso de perfilado, el motor de JavaScript puede identificar qué partes del código pueden mejorarse optimizándolas.
    - Las partes del código que se ejecutan frecuentemente (también conocidas como "hot code") se recompilan utilizando técnicas de optimización avanzadas para mejorar el rendimiento.
    
    Esto puede incluir inlining de funciones, eliminación de código muerto, y otras optimizaciones de bajo nivel.
    
8. **Deoptimization (Reversión):**
    
    Si el motor de JavaScript advierte que una optimización no es válida (por ejemplo, si el tipo de datos esperado cambia), puede “desoptimizar” el código y volver a una versión menos optimizada para garantizar su correcto funcionamiento.
    
---
### **Ventajas de la Compilación JIT**

- **Mejora del rendimiento:**
    
    La compilación JIT mejora el rendimiento gracias a las optimizaciones hechas en tiempo de ejecución, como traducir el código JavaScript fuente a código máquina nativo justo antes de su ejecución, lo que puede resultar en una ejecución más rápida.
    

- **Flexibilidad y adaptabilidad:**
    
    Un compilador JIT puede recopilar métricas en tiempo de ejecución sobre cómo se está comportando el código, lo que permite hacer optimizaciones dinámicas basadas en el comportamiento del programa. Esto permite realizar mejoras de rendimiento incluso en código preexistente y desplegado.
    
    La capacidad de optimizar y desoptimizar el código en tiempo real permite que el motor de JavaScript se adapte a diferentes patrones de uso y datos dinámicos, manteniendo un alto rendimiento.
    
- **Reducción de la sobrecarga de memoria:**
    
    A diferencia de la compilación previa, la compilación JIT solo compila el código que se ejecuta realmente, lo que reduce la huella de memoria del programa. Esto puede ser beneficioso para aplicaciones grandes en las cuales no siempre se ejecutan todas las vías de código, al compilar las partes del código que se ejecutan con mayor frecuencia optimiza el uso de recursos.
    
- **Ejecución temprana:**
    
    La compilación inicial permite que el código comience a ejecutarse rápidamente, sin esperar a que todas las optimizaciones estén completas.
    

En resumen, la compilación en JavaScript, a través de técnicas JIT, permite que el código JavaScript se ejecute de manera eficiente y rápida. Esta combinación de interpretación y compilación adaptativa permite que los motores de JavaScript modernos logren un rendimiento que rivaliza con los lenguajes compilados tradicionales, mientras mantienen la flexibilidad y dinamismo de un lenguaje interpretado.

# Engines (motores)

## ¿Qe son los motores de JavaScript?

Los motores de JavaScript, tmabien conocidos como motores JS o interpretadores de JavaScrip, son programas esenciales para la ejecusion de codigo JavaScript, su principal funcion es convertir el codigo JavaScript escrito por los desarroyadores a un lenguaje que el ordenador pueda entender y ejecutar.

### **Motores de JavaScript y JIT**

Existen numerosos motores de JavaScript, cada uno con sus propias características y enfoques. Algunos de los más populares son:

- **V8 (Chrome, Node.js)**: Utiliza el compilador JIT llamado TurboFan para optimizar el código JavaScript en tiempo de ejecución.
- **SpiderMonkey (Firefox)**: Utiliza compiladores JIT como IonMonkey y Warp para optimizar el código.
- **Chakra (Microsoft Edge):** Ofrece un buen equilibrio entre rendimiento y compatibilidad.
- **JavaScriptCore (Safari)**: También utiliza técnicas de compilación JIT para mejorar el rendimiento del código JavaScript.

### ¿Como funcionan los motores de JavaScript?

El funcionamiento de los motores de JavaScript se puede dividir en etapas:

1. **Análisis léxico:** El motor analiza el código JavaScript para identificar elementos como palabras clave, identificadores y operadores.
2. **Análisis sintáctico:** Se construye una estructura en árbol que representa la organización del código JavaScript.
3. **Compilación:** El código JavaScript se transforma en un lenguaje intermedio que el ordenador puede entender.
4. **Ejecución:** El código intermedio se ejecuta línea por línea, lo que da vida a las acciones y comportamientos descritos en el código JavaScript.

### Optimizacion del rendimiento:

Los motores modernos utilizan diversas técnicas para optimizar el rendimiento, como:

- **Compilación Just-In-Time (JIT):** El código JavaScript se compila en tiempo de ejecución para la arquitectura específica del ordenador, mejorando significativamente la velocidad de ejecución.
- **Recopilación de basura:** El motor libera automáticamente la memoria utilizada por objetos que ya no son necesarios, evitando fugas de memoria y mejorando el rendimiento general.

### Conclusion:

Los motores de JavaScript juegan un papel fundamental en la ejecución de código JavaScript en navegadores web, entornos de servidor y otras aplicaciones. Su constante evolución y optimización permiten que las experiencias web y las aplicaciones JavaScript sean cada vez más rápidas, dinámicas y seguras.
